# 🎁 Fable5 vs GPT-5.6 比較用プロンプト100選（非エンジニア向け）

同じ1本を Claude（Fable 5）と ChatGPT（GPT-5.6）の両方に貼って、返ってくる答えを見比べるための100本です。カテゴリごとに10本ずつ、全10カテゴリ。用途に合う「相棒」を見つける物差しに使ってください。

**使い方**: 気になるプロンプトをコピー → まず片方に貼って実行 → もう片方にも同じものを貼って実行 → 出力の「深さ・速さ・語り口・過不足」を見比べる。〔　〕は自分の状況に置き換えてください。

## 1. 文章を書く・整える

このカテゴリでは、同じ「気まずい」「言いにくい」場面でも、両モデルが選ぶ言葉遣いの柔らかさや、相手への配慮の入れ方にどんな違いが出るかを見てください。

### 1. 取引先への謝罪メール
比べどころ: 謝罪の言葉選びと、言い訳がましくならない配慮の違い。

```text
取引先に納期を3日遅らせてほしいと伝える謝罪メールを書いてください。
理由は〔社内の確認作業に時間がかかったため〕です。
・言い訳がましくならないこと
・今後の対応策を一言添えること
・ビジネスメールとして自然な長さにすること
件名も含めて出力してください。
```

### 2. 遅刻・欠席の連絡メール
比べどころ: 簡潔さと丁寧さのバランスをどちらがうまく取るか。

```text
明日の〔会議/授業参観〕に、〔体調不良〕のため参加できなくなったことを伝える連絡文を作ってください。
相手は〔上司/学校の先生〕です。
・お詫びと今後の対応（資料を後で確認する等）を1文ずつ入れる
・全体で5行以内におさめる
```

### 3. 上司への相談メール（言いにくい内容）
比べどころ: 「言いにくいこと」をどこまで踏み込んで代弁してくれるか、遠慮の度合いの違い。

```text
上司に、今担当している〔プロジェクト名〕の業務量が多すぎて困っていることを相談したいです。
・不満っぽく聞こえないようにする
・改善してほしい点を2つに絞る
・最後に「相談の時間をもらえるか」と一言添える
このトーンでメール文を作ってください。
```

### 4. お礼メール（初対面の相手へ）
比べどころ: 儀礼的な定型文で終わるか、相手固有のエピソードを拾って個性を出すか。

```text
先日〔異業種交流会〕で初めてお会いした〔名前・会社名〕さんに、お礼のメールを送りたいです。
話した内容は〔新しい働き方についての雑談〕でした。
形式的になりすぎず、また会いたいと思ってもらえるような一言を入れて書いてください。
```

### 5. 結婚式のスピーチ原稿
比べどころ: エピソードの盛り込み方や感動的な言い回しの作り込み度合い。

```text
友人の結婚式で読む3分程度のスピーチ原稿を作ってください。
新郎（新婦）とは〔大学のサークル仲間で10年来の友人〕です。
エピソードとして〔学生時代に一緒に旅行に行った話〕を入れたいです。
明るく、少し笑いも取れるトーンでお願いします。
```

### 6. 自己紹介文（副業/転職用）
比べどころ: 強みの言語化のうまさと、誇張しすぎない自然な自己PRのさじ加減。

```text
副業でWebライターとして活動するためのプロフィール文を作ってください。
経歴は〔本業は事務職・文章を書くのが好きでブログを2年続けている〕です。
・実績を誇張しない
・依頼したくなるような信頼感を出す
200字程度でお願いします。
```

### 7. SNS投稿文（宣伝っぽくならない紹介文）
比べどころ: 「宣伝臭さ」を消す言葉選びの上手さの差。

```text
自分が作った〔手作りアクセサリー〕をInstagramで紹介する投稿文を作ってください。
・売り込み感を出さず、日常の一コマとして自然に見せる
・ハッシュタグを5個提案する
・絵文字は使いすぎない
```

### 8. クレーム対応の返信文
比べどころ: 相手の感情を受け止める初動の書き方と、事実確認の伝え方の違い。

```text
お客様から〔届いた商品に傷があった〕というクレームが届きました。
・まず謝罪し、相手の気持ちに寄り添う
・状況確認のために必要な情報（購入日・写真など）を丁寧にお願いする
・次のアクションを明示する
この3点を満たす返信文を作ってください。
```

### 9. 依頼を断るメール
比べどころ: 関係を壊さずに断る言い回しの巧みさ。

```text
知人から頼まれた〔無償での仕事の手伝い〕を断りたいです。
・関係を悪くしたくない
・角が立たない理由の伝え方にしたい
・今後も付き合いを続けたい旨を添える
このニュアンスでメール文を作ってください。
```

### 10. 長文メールの要点だけを抜き出して短縮
比べどころ: 情報を削るときに何を残し何を捨てるかの判断基準の違い。

```text
以下の長いメールを、要点だけ3行に要約してください。相手に返信する前に内容を素早く把握したいです。

〔ここに実際の長いメール本文を貼り付ける〕

要約のあとに「返信で確認すべきこと」を箇条書きで2〜3個挙げてください。
```

## 2. 要約する・調べる

正確さへの慎重さ（断定を避けるか／根拠を示すか）と、専門用語のかみ砕き方の違いに注目してください。

### 11. 契約書・規約の要注意点の洗い出し
比べどころ: 「専門家に確認を」と留保をどれくらい入れるか、注意点の粒度の違い。

```text
以下は〔賃貸契約書/業務委託契約書〕の一部です。
素人が見落としがちな注意点を、やさしい言葉で5つ挙げてください。
最終的な判断は専門家に確認する前提で、あくまで「気づきのヒント」として教えてください。

〔ここに契約書の該当部分を貼り付ける〕
```

### 12. ニュース記事の3行要約＋背景説明
比べどころ: 事実と背景（推測）を分けて書くかどうかの誠実さの違い。

```text
以下のニュース記事を3行で要約してください。
そのあとに、このニュースの背景を知らない人にもわかるように5行程度で補足説明を加えてください。
事実部分と、記者・専門家の見方（推測）は分けて書いてください。

〔ここに記事本文またはURLを貼り付ける〕
```

### 13. 専門用語（保険/税金など）をやさしく解説

```text

・日常生活のたとえを1つ入れる

```

### 14. 2つの商品・サービスの比較表作成

```text

表のあとに、この条件ならどちらが向いているか、理由つきで一言まとめてください。
```

### 15. 長い会議の音声書き起こしの要約

### 16. 業界動向の初心者向け解説

## 3. 企画・アイデア出し

アイデアの「量」で押すのか「筋の良さ」で絞るのか、発想の広げ方の癖の違いが出やすいカテゴリです。
